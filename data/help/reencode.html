인코딩

- 비디오 해상도를 변경하거나 오디오 샘플 레이트를 변경하려 할 때 간단하게 사용할 수 있습니다.

- 동영상을 하나로 잇기 할 때도 인코딩이 필요해집니다. 왜냐하면, 두 파일 간에 다음 요소들이 일치해야 하기 때문입니다.

  • ·비디오의 해상도, 타임베이스, 픽셀 형식, 코덱
  • ·오디오의 샘플 레이트, 채널, 코덱

- '다른 파일 참조...' 버튼은 다른 파일의 속성(해상도, 타임베이스, 픽셀 형식, 코덱, 샘플 레이트, 채널) 값을 그대로 적용하려 할 때 사용합니다.

- '최근 인코딩 적용' 버튼은 최근에 실행했던 인코딩 작업을 그대로 따라 하려 할 때 사용합니다.

타임베이스

- 최신 컨테이너(MKV, MP4, MOV 등)는 프레임 속도가 아닌 타임스탬프에 기반한 시간(pts_time)을 사용하여 비디오 프레임 표시의 시간 구성 요소를 제어합니다. 따라서 비디오를 25fps(=초당 25프레임)로 녹화하여 각 프레임을 0.04초 간격으로 그려야 함을 암시하는 대신 각 프레임에 대한 타임스탬프를 저장합니다. 

 Frame      pts_time
   0          0.00
   1          0.04
   2          0.08
   3          0.12
   ...

- 이러한 시간 값을 정확하게 분석하기 위해 타임베이스(timebase), 즉 시계의 한 틱(tick)을 나타내는 시간 단위가 사용됩니다. 따라서 1/75의 타임베이스는 1/75초를 나타냅니다. 그런 다음 프레젠테이션 타임스탬프(pts)는 이 타임베이스를 기준으로 표시됩니다. 시간 척도(timescale)는 단순히 시간 기준의 역수입니다. FFmpeg는 스트림 판독에서 시간 척도를 tbn 값으로 표시합니다 

Timebase = 1/75; Timescale = 75
 Frame        pts           pts_time
   0          0          0 x 1/75 = 0.00
   1          3          3 x 1/75 = 0.04 
   2          6          6 x 1/75 = 0.08
   3          9          9 x 1/75 = 0.12
   ...
